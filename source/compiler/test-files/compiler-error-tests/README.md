Most compilation errors can be elicited by asking Pipefish to compile a single expression, maybe in the context of a script. Errors that can only happen *in* a script would usually be erors in the initializer, where they'd actually happen.

However, there are some errors which are generated by the compiler but only make sense in the context of running a script, e.g. those involving the `global` keyword. We deal with those as in the `initialization-error-tests` folder, by having a file for each error to be elicited, which is named after it.
